Guía Rápida de Git

Conceptos fundamentales para el control de versiones.

¿Qué es Git?

Git es un sistema de control de versiones distribuido que permite rastrear cambios en el código fuente.

Comandos Básicos

  • git init: Inicia un repositorio.
  • git add: Prepara archivos.
  • git commit: Guarda cambios.

Flujo de Trabajo

El flujo típico consiste en: Modificar -> Stage (add) -> Commit -> Push.

Preguntas Frecuentes (FAQ)

¿Por qué usar ramas? Permiten trabajar en funciones nuevas sin romper el código que ya funciona en la rama principal.

¿Qué es un Commit? Es como una "foto" o captura del estado de tus archivos en un momento exacto.
